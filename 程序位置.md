# mdedit 窗口位置管理

## 概述

mdedit 的窗口位置管理覆盖以下场景：

| 场景 | 命令/行为 | 效果 |
|------|-----------|------|
| 默认启动 | `mdedit` | 恢复上次关闭时的位置和大小 |
| 重置窗口 | `mdedit --reset` | 窗口重置为 1200×800，主屏居中 |
| 指定位置 | `mdedit --setpos 3840,222` | 窗口定位到指定物理像素坐标 |
| 最大化恢复 | 配置中 `maximized=true` | 以最大化状态启动 |

---

## 配置存储

### 文件位置

`%APPDATA%/mdedit/config.ini`（INI 纯文本格式）

### 窗口相关字段

```ini
window_x=1234.0        # 窗口左上角 X（物理像素）
window_y=567.0         # 窗口左上角 Y（物理像素）
window_width=1200.0    # 窗口宽度（egui points）
window_height=800.0    # 窗口高度（egui points）
maximized=false        # 是否最大化
```

### 坐标单位说明

- `window_x` / `window_y`：**物理像素**（Physical Pixels），不受 DPI 缩放影响
- `window_width` / `window_height`：**egui points**（逻辑像素），受 DPI 缩放影响
- 两者混用的原因：`outer_rect` 返回 points 需乘以 ppp 转物理像素，而 `inner_rect` 和 `with_inner_size` 都使用 points

## 核心流程

### 一、启动位置恢复

```
main() 入口
  ├── SetProcessDpiAwarenessContext()     ← 设置 Per-Monitor DPI V2
  ├── 解析命令行: --reset / --setpos / 文件路径
  ├── AppConfig::load()                   ← 从 config.ini 加载配置
  ├── log_monitors()                      ← 记录所有显示器信息
  └── 确定 target_pos
        ├── reset=true         → None（不指定，系统默认居中）
        ├── setpos=Some(x,y)   → Some((x,y))（命令行指定）
        ├── cfg有保存的坐标     → 验证 is_position_visible → Some/None
        └── 其他               → None
```

**关键代码**（`src/main.rs:296-304`）：

```rust
let target_pos: Option<(f32, f32)> = if reset {
    None
} else if let Some(pos) = setpos {
    Some(pos)
} else if let (Some(x), Some(y)) = (cfg.window_x, cfg.window_y) {
    if is_position_visible(x, y, w, h) { Some((x, y)) } else { None }
} else {
    None
};
```

`target_pos` 通过 `MdEditApp::new()` 传入应用实例。

### 二、延迟定位（第 2 帧）

窗口位置不在创建时设置，而是延迟到**第 2 帧**，原因：

1. 第 1 帧 egui 上下文尚未完全初始化
2. DPI 缩放因子 `pixels_per_point()` 需要第 1 帧后才准确
3. 窗口管理器可能尚未完成布局

**关键代码**（`src/app.rs:1082-1088`）：

```rust
if self.frame_count == 2 {
    if let Some((px, py)) = self.target_physical_pos.take() {
        let ppp = ctx.pixels_per_point();
        let pos = egui::pos2(px / ppp, py / ppp);  // 物理像素 → egui points
        ctx.send_viewport_cmd(egui::ViewportCommand::OuterPosition(pos));
    }
}
```

### 三、退出时保存

**关键代码**（`src/app.rs:1877-1922`）：

```rust
// 每帧追踪窗口状态
fn update(&mut self, ctx: &egui::Context, ...) {
    let ppp = ctx.pixels_per_point();
    ctx.input(|i| {
        if let Some(rect) = i.viewport().inner_rect {
            self.last_window_size = Some((rect.width(), rect.height()));  // points
        }
        if let Some(rect) = i.viewport().outer_rect {
            self.last_window_pos = Some((rect.min.x * ppp, rect.min.y * ppp));  // 转物理像素
        }
        self.last_maximized = i.viewport().maximized.unwrap_or(false);
    });
}

// save() 和 on_exit() 都会调用 save_config()
fn on_exit(&mut self) {
    self.save_config();  // 写入 config.ini
}
```

## 多显示器处理

### 显示器枚举

`log_monitors()`（`src/main.rs:60-124`）通过 Win32 API `EnumDisplayMonitors` 枚举所有显示器，记录：

- 显示器句柄
- 屏幕区域坐标（`rc_monitor`）
- 工作区坐标（`rc_work`，排除任务栏）
- DPI 和缩放比例
- 是否主屏

### 位置可见性检测

`is_position_visible()`（`src/main.rs:130-150`）通过 `MonitorFromRect` API 检测指定矩形是否属于某个显示器：

```rust
fn is_position_visible(x: f32, y: f32, w: f32, h: f32) -> bool {
    // 构造 RECT，调用 MonitorFromRect(&rc, MONITOR_DEFAULTTONULL)
    // 返回 monitor != 0（0 表示不属于任何显示器）
}
```

**作用**：当用户在双屏环境下使用 mdedit，断开外接显示器后再次启动，窗口不会消失在"虚空"中，而是回退到系统默认位置。

### 非 Windows 平台

使用简单的范围判断替代：

```rust
fn is_position_visible(x: f32, y: f32, _w: f32, _h: f32) -> bool {
    x >= 0.0 && y >= 0.0 && x < 5000.0 && y < 3000.0
}
```

## 命令行参数总结

| 参数 | 格式 | 示例 | 说明 |
|------|------|------|------|
| 无参数 | `mdedit` | | 恢复上次位置，首次启动使用系统默认 |
| `--reset` | `mdedit --reset` | | 重置为 1200×800，主屏居中，不保存位置 |
| `--setpos` | `mdedit --setpos x,y` | `mdedit --setpos 3840,222` | 指定物理像素坐标，优先级最高 |
| `--debug-theme` | `mdedit --debug-theme light` | | 调试模式，输出主题解析后退出 |

优先级：`--reset` > `--setpos` > 配置文件恢复 > 系统默认
